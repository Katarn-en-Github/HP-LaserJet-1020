# 02. HP LaserJet 1020 en Raspberry Pi OS Trixie

## Objetivo

Este documento describe el procedimiento completo para restaurar el
funcionamiento de la **HP LaserJet 1020** en una instalación limpia de
**Raspberry Pi OS Trixie** con **CUPS 2.4.10**.

------------------------------------------------------------------------

# Historia del problema

La HP LaserJet 1020 pertenece a una generación de impresoras que **no
almacenan permanentemente su firmware**.

Cada vez que se enciende debe recibir dicho firmware desde el sistema
operativo.

En versiones antiguas de Linux este proceso era transparente, pero
actualmente HP ya no distribuye el firmware como parte del sistema
operativo y la impresora suele aparecer instalada pero no imprime.

------------------------------------------------------------------------

# ¿Por qué falla en Linux moderno?

Normalmente ocurre porque falta:

-   `sihp1020.dl`
-   el script `hplj1020`
-   el controlador `printer-driver-foo2zjs`

Sin el firmware la impresora nunca cambia al modo operativo.

------------------------------------------------------------------------

# Explicación del firmware

El paquete foo2zjs descarga desde HP un archivo comprimido
(`sihp1020.tar.gz`) que contiene:

-   `sihp1020.img`
-   `sihp1020.dl`

El archivo importante para Linux es:

    /lib/firmware/hp/sihp1020.dl

Cuando la impresora se conecta, el script de udev lo envía
automáticamente al dispositivo.

------------------------------------------------------------------------

# Archivos necesarios

Desde este repositorio se requieren:

    firmware/
        sihp1020.dl
        sihp1020.img
        sihp1020.tar.gz

    ppd/
        HP-LaserJet_1020.ppd

    scripts/
        hplj1020

    source/
        foo2zjs_20200505dfsg0.orig.tar.xz
        foo2zjs_20200505dfsg0-3.debian.tar.xz
        foo2zjs_20200505dfsg0-3.dsc

------------------------------------------------------------------------

# Dónde copiar cada archivo

Firmware:

    sudo mkdir -p /lib/firmware/hp
    sudo cp sihp1020.dl /lib/firmware/hp/

Script:

    sudo cp hplj1020 /usr/lib/udev/
    sudo chmod 755 /usr/lib/udev/hplj1020

El archivo PPD puede mantenerse en cualquier carpeta del usuario e
importarse desde la interfaz web de CUPS.

------------------------------------------------------------------------

# Instalación desde cero

1.  Instalar CUPS.
2.  Instalar:

``` bash
sudo apt install printer-driver-foo2zjs
```

3.  Copiar el firmware.

4.  Copiar el script.

5.  Reiniciar:

``` bash
sudo reboot
```

6.  Entrar a:

Usando cualquier navegador WEB que se encuentre conectado a la misma RED:
<!-- -->
```
    http://IP_DE_LA_PI:631

Agregar la impresora USB utilizando el archivo PPD.

------------------------------------------------------------------------

# Comprobaciones

Verificar firmware:

``` bash
ls -l /lib/firmware/hp/sihp1020.dl
```

Verificar script:

``` bash
ls -l /usr/lib/udev/hplj1020
```

Verificar paquete:

``` bash
dpkg -l | grep foo2
```

Imprimir una página de prueba desde CUPS.

------------------------------------------------------------------------

# Solución de problemas

## Error "Filter failed"

Generalmente indica firmware inexistente.

------------------------------------------------------------------------

## La impresora aparece pero no imprime

Verificar:

-   firmware
-   foo2zjs
-   hplj1020

------------------------------------------------------------------------

## No hay actividad en la impresora

Comprobar:

``` bash
journalctl -b | grep hplj1020
```
